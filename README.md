# 🍓 Strawberries Marketplace
Распределенная микросервисная система для онлайн-маркетплейса. Проект разрабатывается в рамках изучения сервис-ориентированной архитектуры (SOA) и современных технологий на стеке Java/Spring.

## Архитектура
Система построена на принципах микросервисного взаимодействия с разделением на контракты (API) и реализации (service).

### Текущие микросервисы:
1. User Service — управление пользователями, ролями и профилями.
    * Протокол: REST + HATEOAS.
    * БД: PostgreSQL.

### Планируемые микросервисы:
* Product Service (GraphQL/DGS) — каталог товаров и склад.
* Order Service (REST/Events) — управление заказами и статусами.
* Payment Service — имитация платежных операций.

## Технологический стек
* Java 21
* Spring Boot 4.0.5
* Gradle (мультимодульная сборка)
* Spring Data JPA & Hibernate
* PostgreSQL (хранение данных)
* MapStruct (типобезопасный маппинг)
* Lombok (сокращение шаблонного кода)
* Docker & Docker Compose (контейнеризация)
* SpringDoc OpenAPI (Swagger) — документация API

## Структура проекта
Проект использует паттерн API-first (разделение на интерфейсы-контракты и логику):

```text
Strawberries/
├── api/                # Общие интерфейсы и DTO (библиотеки)
│   └── user-api/       # Контракт User Service
├── service/            # Реализации микросервисов (приложения)
│   └── user-service/   # Исполняемый сервис пользователей
└── docker-compose.yml  # Запуск всей инфраструктуры 
```
## Быстрый запуск
1. Требования:
2. Docker и Docker Compose
3. Java 21 (для локальной разработки)

### Запуск всей системы:
В корневой папке проекта выполните команду:

```bash
docker-compose up --build
```
После запуска сервис будет доступен по адресу: http://localhost:8080

### Документация API:
Swagger UI доступен по адресу: http://localhost:8080/swagger-ui/index.html

## Особенности реализации
1. User Service 
   * Soft Delete: Пользователи не удаляются физически, а помечаются флагом active.
   * Частичное обновление: Поддержка метода PATCH через MapStruct.
   * HATEOAS: Ответы API содержат навигационные ссылки (Self-links, Paging links).
   * RFC 7807: Стандартизированные ответы об ошибках (Problem Details for HTTP APIs).
   * Database Constraints: Уникальность email только для активных пользователей через частичные индексы PostgreSQL.
